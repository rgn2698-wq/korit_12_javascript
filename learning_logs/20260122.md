# JavaScript의 역사
1990년도에 브랜던 아이크가 처음에 모카(Mocha)라는 이름으로 지었었음. 그 다음에 livescript로 갔다가 javascript가 되었음. 기본적으론 브라우저에서 실행되는 프로그래밍 언어임.

옛날에는 각 브라우저 상에서 구동되는 언어가 달랐기 떄문에 A브라우저에서 실행되는 사이트가 B브라우저에선 안되는 일이 자주 발생해서 Script 언어에 대한 표준이 필요해졌음

# ECMA Script
European Computer Manufacturers Association 이라는 유럽 컴퓨터제조업체 연합에서 JavaScript의 표준화에 대한 작업이 시작되었음. ECMA-262으로 시작했었음. 즉 JS 자체가 아니라 다른언어도 가능함. 하지만 사실상 JS를 표준화 하기위해서 만들어진 스크립트 표준임. ECMA의 축약어가 ES고 ES6

# JS문법
## 변수와 상수
- 변수(Variable) - 데이터를 담는 바구니 : 데이터/객체/함수를 사용하겠다고 선언하고 이름을 붙이는것.
- 상수(Const) - 변화하지않는 값을 가지는 변수

## 변수 언선 현식

```js
var x = "Hello World!";
선언자 변수명 = 데이터
```

### 변수 명명 규칙
1. 문자, 숫자, _, $를 포함하여 만들수 있다.
2. 변수명은 숫자로 시작할 수 없다,.
3. JS예약어는 사용할수 없다. (console과 같은 변수명으로 쓸수없음)
4. 변수명은 대소문자를 구분한다.

- ES에서 변수명과 함수명은 Camel case 사용 / 클래스명과 생성자명에는 Pascal case를 권장.

### 선언자의 종류
1. var    - 변수용
2. let    - 변수용
3. const  - 상수용

- var 선언자
  - JS는 Java와는 달리 데이터 타입에 상관없이 변수를 사용할때 var / let을 사용할수 있음. 현재에는 오류를 최소화 하기 위해 주로 let을 사용하는데 이미 개발된 많은 JS 프로그램이 var로 되어있기 떄문에 혹시 유지보수 팀으로 들어가게 될 경우 var선언자를 보게될 확률이 높음.

```js
var x = 5
var y = 6
var z = x + y
```

02_variable.js

```js
var x = 5
var y = 6
var z = x + y
console.log(z)

var x = 7;    // 여기서 오류가 발생해야 정상인것 같음.
z = x + y
console.log(z)
```

그런데 오류가 발생하지않는 점을 봤을때 var 선언자를 사용하게 되면 변수명이 겹치더라도 오류가 나지않기 때문에 원하는 결과값이 다르게 나올수도 있음.

동일한 변수명으로 새로운 변수가 선언되면 기존 코드에서 동일한 이름으로 사용되던 변수에 할당된 값이 사라지므로, 해당 변수를 참조하고 있는 프로그램에서 오류가 발생할 가능성이 있음.

그래서 ES6 이후로 let 선언자가 생겼고, 오늘날에는 let을 쓸것을 권장하는 중.

- let 선언자
동일한 변수명을 허용하지도않고 나중에 선언된 변수를 미리 등록해두지도 않음.

- const 선언자
final과 같긴 함. 선언시점에 값을 할당한 후에 변수로 재선언 및 재 대입이 불가능
- java의 경우에 final 키워드는 되게 조심스럽게 사용했지만 js의 경우는 좀 더 적극적으로 const를 사용함.

- JS의 개발철학상 변수는 브라우저에서 값을 바꿀 수 있기 때문에 모두가 동일한 결과 값을 가지기가 힘듦. 하지만 상수로 선언하게 되면 값이 바뀌지 않기 때문에 다수의 사용자가 동일한 결과값을 얻을 수 있다는 점 때문에 const의 사용을 장려함.

그래서 함수는 const로 선언하는 편

* 주의 : 함수의 결과값을 const로 집어넣는다는 의미는 아님

- string
```js
let name1 = "김일";
let name2 = '김이';

let singleQuote = "그의 이름은 '김일'입니다,";
let doubleQuote = '그의 이름은 "김이"입니다.";
```
표준적 으로는 String입력할때 ' '를 쓰는편
- HTML에서 태그 내에 값 입력할때 " "가 나왔기 때문에 헷갈리지 않도록 ''를 쓰는 편임.
- Number
숫자 타입에 대한 구분이 없음 그냥 따음표 없이 쓰면 됨.
- Boolean
- Undefined
  - undefined 자체가 데이터 값이자 데이터 자료형임. 단어 그대로 정의 하지않았다는 의미

```js
let car;    //선언은 했지만 아직 초기화가 되지 않았으면
console.log(car);   // undefined로 콘솔에 찍힘
```

- null 
  - 똑같이 데이터 값이자 자료형. 없음을 의미하는 것은 동일한데 의미상의 차이가 존재해서 구분을 해줘야함.
    - undefined의 경우에는 웹 브라우저에서 JS를 실행할때 초기화가 되지않은 변수에 대해 자동으로 undefined를 리턴해주기 때문에 개발자가 아니라 브라우저 엔진이 할당하는 값임.
    - 반면에 null의 경우는 개발자가 명시해서 사용함. 아직 어떤값을 할당할지 모르거나 어떤 자료형의 값이 들어올지 모를때 개발자가 명시적으로 null을 대입해둘수 있음.
```js
let car = null;
```
  - null은 JS의 메모리를 효율적으로 관리하고 싶을때 사용할수 있음.
  - 일반 변수를 선언해서 값 집어넣고있다가, 더이상 해당 변수가 사용되지 않을때 맨 마지막으로 null 값을 넣어버리게 되면 메모리에서 해당 변수를 더이상 참조하지않게 됨.
  - 그러면 더이상 안쓰이는 변수에 대한 메모리 누수가 존재하지않게 된다는 점에서 특정시점에 null값을 대입해주는 방식을  통해 사용성을 향상 시킬수 있음.

- Symbol - ES6에 추가된 자료형

## Object
### Objects
- 추후 좀 자세히 다룰예정
- 기본 적으로 JS의 모든 것은 객체에 해당함. 그런데 선언방식이 좀 다름.
```js
const person = {
  firstName: 'john',
  lastName: 'Doe'
  age: 50,
  eyeColor 'blue'
}
```
key
  - value properties로 이루어져있다 라고 했었음.

```js
person["age"] = 51; // 객체명 ["key값"] = 재대입할 데이터;
person.age = 51;  // 객체명.key값 = 재대입할 데이터;

console.log(person['firstName']);
console.log(person.firstName);
```
### Array
= 하나 이상의 데이터를 하나의 변수에 담아뒀음. 여러 값을 하나의 단일 참조로 저장할수있음.(index를 통해) 배열에 저장된 데이터의 특징은 순서를 갖는다는 점.

```js
const cars = ['현대', '기아', '혼다'];

console.log(cars[0]);
console.log(cars[1]);
console.log(cars[2]);

cars[2] = '벤츠';
```

그러면 초기화 한 값에서 추가하거나 혹은 삭제하는 방법에 대한 문제가 생길수있음. 이는 배열 내장함수를 통해서 학습.

## typeof
- JS에서는 typeof 연산자를 활용하여 변수의 자료형을 알아낼수 있음.
- Java에서는 필요없었음. 애초에 선언할때 우리가 알고 선언해야 하기떄문

```js
let name = '김일';
console.log(typeof name);
```

instanceof의 경우에는 a instanceof b 의 형태로 써야만 했음 하지만 하나만 쓴다는 점에서 연산자라고 생각하기 어려울수 있지만 함수라고 이야기 하지않는 이유는 typeof(name); 이 아니기 떄문.
